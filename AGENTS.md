## Stack
Go (primary language); Go modules for dependency management; Docker for containerization. No web framework detected.

## Constraints
Never modify:
- `go.sum` (lock file — managed by `go mod tidy` only)
- Any `*_generated.go` or `zz_generated_*.go` files
- Any file named `*.pem`, `*.key`, `*.crt`, or containing credentials/secrets
- `Dockerfile` base image digests (do not pin or change `FROM` digest lines without explicit instruction)
- Migration files in any `migrations/` directory

## Conventions
- All Go source files live at repo root or in subdirectories; follow existing package structure
- Test files use the `_test.go` suffix and must be in the same package as the code under test
- Exported functions, types, and methods must have Go doc comments (`// FuncName ...`)
- `go mod tidy` must pass cleanly after any dependency change
- Docker images must pass `hadolint` without new warnings after any Dockerfile edit

## Dependency manifests
- `go.mod` — Go module dependencies and minimum versions
- `go.sum` — DO NOT edit manually; regenerated by `go mod tidy`
