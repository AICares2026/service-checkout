## Stack
Go (primary language), Docker. Go modules for dependency management. No framework detected beyond standard library.

## Constraints
Never modify:
- `go.sum` (lock file — managed by `go mod tidy` only)
- Any file matching `*_generated.go` or `*.pb.go`
- `Dockerfile` base image pins unless fixing a confirmed CVE
- Credential or secret files: `*.pem`, `*.key`, `*.env`, `.env*`
- Migration files: any directory named `migrations/` or files matching `*.sql`

## Conventions
- Tests live alongside source files, named `*_test.go`
- Exported functions must have test coverage — add `TestXxx` functions in the same package directory
- One package per directory; follow standard Go project layout
- Docker image tags and build args are defined in `Dockerfile` at repo root
- `go mod tidy` must be run after any change to imports or `go.mod`

## Dependency manifests
- `go.mod` — module path, Go version, direct and indirect dependencies
- `go.sum` — **read-only**; regenerated by `go mod tidy`, never hand-edited
