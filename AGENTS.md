## Stack
Go (primary language); Go modules for dependency management; Docker for containerization.

## Constraints
Never modify:
- `go.sum` (lock file — managed by `go mod tidy` only)
- Any `*_generated.go` or `zz_generated_*.go` files
- `Dockerfile` lines containing credentials, secrets, or `ARG`/`ENV` entries with tokens
- Migration files matching `migrations/` or `*_migration.go`
- `.env`, `*.pem`, `*.key`, `*.crt`, credential or secret files

## Conventions
- Test files end in `_test.go` and live alongside source files (same package directory)
- Exported functions without tests are candidates for coverage addition (`go_test_coverage` skill)
- Docker images are defined in `Dockerfile` at repo root; lint fixes applied by `dockerfile_lint_fix`
- `go mod tidy` must be run after any dependency change; never edit `go.sum` directly
- CVE scan uses version `d6cc81e5`; do not alter scan configuration or version pins

## Dependency manifests
- `go.mod` — module path, Go version, direct and indirect dependencies
- `go.sum` — cryptographic hashes (read-only; regenerated by `go mod tidy`)
